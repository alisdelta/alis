# ALIS_math_v0.2.1_public_alignment_JP
## 公開版との対応・補足メモ（v0.2.1 更新版）

この文書は、**ALIS 数学的骨格 v0.2.1** を公開版ALISと接続して読むための補足である。  
数理版は公開版を上書きするものではなく、公開版の最小骨格を形式化の方向から点検する補助文書として置かれる。

v0.2 では主に **A3 / A4 / E1** を最小差分で引き締めた。  
v0.2.1 では、v0.2 を前提として **A4 周辺の最小精緻化** のみを行った。狙いは、

- A4 の収束の測度的意味を、A1 の `P` と整合する形で明示すること
- 回復作用族 `\mathcal{R}_c` 自体が時間とともに縮退する過程もA4の射程に含まれることを明示すること

の2点に絞られる。

また、v0.2 で一度検討された **B100応答境界に関する節（旧E2案）** は、
数理版本体には挿入せず、独立別紙
「**ALIS応答境界論 v0.1（External Response Boundary）**」として切り出された。

## 2. 補足が必要な語の自然語説明

### `P(\Delta)`
差分 `\Delta` が、どれだけ**まだ観測できるか・再展開できるか・分岐可能性を残しているか**を測る量。  
公開版の感覚では、「差分がまだ死んでいないか」を見るための補助記号に近い。

### `\mathcal{R}_c`
文脈 `c` において、失われかけた差分や多声性を**回復・再開・再参照するために許される操作の集まり**。  
たとえば、時間を置く、履歴を再読する、別の文脈から見直す、といった操作がここに入る。

### `\mathcal{R}_c^{(t)}`（v0.2.1追加）
時刻 `t` の時点で実際に利用可能な回復作用の集まり。  
`\mathcal{R}_c` そのものは抽象的な族だが、現実の運用では、ある時点で使える回復経路の集合は
時間とともに変わりうる。

たとえば、

- 時間が経ちすぎて履歴へのアクセスが実質的に閉じた
- 別視点を与えてくれた人との関係が途絶えた
- ある文脈自体が社会的・内面的に失われた

といった状況では、当初使えた回復作用がもはや使えなくなっている。  
この「時刻ごとに使える回復作用の族」を区別して書くための記号である。

### `\mathcal{R}_c^{(t_1)} \supseteq \mathcal{R}_c^{(t_2)}`（v0.2.1追加）
時刻 `t_1` から `t_2`（ただし `t_1 < t_2`）にかけて、利用可能な回復作用の集合が**単調に縮んでいく**ことを表す。  
「昔は戻れた道が、時間が経つにつれて一つずつ塞がっていく」状況を形式的に書くための表現。

v0.2.1 では、このような縮退が進むと、当初は soft collapse に見えた状態が、
いずれ hard collapse に移行しうることを、A4 の射程に含めることになる。

### `S_c`
文脈 `c` において、**その場で本当に保護対象とみなす差分の候補集合**。  
すべての差分を一律に扱うのではなく、「この文脈では何が意味のある差分か」を切り出すために導入された。

### `\eta(c)` と `\varepsilon(c)`
- `\eta(c)`：文脈 `c` において、「これ以上なら意味のある差分とみなす」ための下限。  
- `\varepsilon(c)`：文脈 `c` において、「ここまで落ちると実質的に回復不能とみなす」ための上限。

v0.2 では

```math
0\le \varepsilon(c) < \eta(c)
```

と置くことで、**最初からほとんど差がないもの**と、**意味のある差分を潰した場合**を切り分けている。

### A3 で言っていること
A3 は、**もともと意味のある差分だったものを、どんな回復操作をかけても死にかけライン以下に落としてしまうこと**を禁じている。  
公開版でいう「Δ圧縮禁止」を、少しだけ数理的に言い換えたもの。

### `V(A)=1`
集合 `A` が、実効的には**単一枝しか支持していない**ことを表す記号。  
公開版でいう「単一波形化」「単一の正解への閉鎖」に近い感覚を、状態空間側で受けるための印である。

### A4 で言っていること
A4 は、広い領域の状態が、反復作用の結果として**長期的に単一枝へ押し込まれ、しかも回復できない**ことを禁じている。  
これは公開版の「単一波形禁止」を、1回きりの圧縮ではなく**長期挙動としての閉鎖禁止**に寄せて読ませるための補強である。

v0.2.1 では、ここに**時間的縮退による回復不能化**も含まれることを明示した。

### A4 の収束の測度的意味（v0.2.1 補足）
A4 の式 `R(T^n(x))\to A` における「収束」は、A1 で採用した保存量 `P` に**整合する意味**で読む。

- 距離空間版 `P_T` を採用した場合 → 距離収束として読む
- 情報理論版 `P_T^{\mathrm{info}}` を採用した場合 → 情報量収束として読む

「何がどう近づいていくか」は、選んだ `P` の種類に依存する。  
この点を明示的に書いておくことで、A4 の運用時にどの収束概念で読むかがぶれない。

### 劣化 collapse（v0.2.1追加）
静的に見ると soft collapse だが、時間とともに `\mathcal{R}_c^{(t)}` が痩せていき、
結果として hard collapse に移行するタイプの閉鎖。

実運用上、これがもっとも危険なパターンに近い。なぜなら、

- その時点で見ると「まだ戻れる」に見える
- 時間が経ってから「実はもう戻れない」と気づく

という**事後的な発覚**を含むからである。  
v0.2.1 では、これも A4 の禁じる閉鎖の一種として位置づけられている。

### `\mathcal{D}_c^{\mathrm{red}}(\Psi)`
状態 `\Psi` を、文脈 `c` のもとで**どのような複数モードの重なりとして読んでよいか**を与える、**簡約許容分解族**。  
前版の `\mathcal{D}_c(\Psi)` より一段厳しく、

- 構造保存的
- 非冗長
- 有効モードのみを数える

という条件を入れている。

### `a_{\min}(c)`
モードの振幅が、少なくともここまではないと**有効モードとして数えない**ための下限。  
ごく小さい揺らぎを、都合よく「別モード」と数えてしまうことを防ぐ。

### `p_{\min}(c)`
そのモードが保持している差分保存量に関する下限。  
振幅だけあっても、差分をまったく保持していないモードは、ALIS 的には有効とみなさないために置かれる。

### `r_D` と `\rho(c)`
- `r_D`：分解 `D` における残差項
- `\rho(c)`：その残差が「十分小さい」とみなせる上限

これにより、「主要なモード分解」と「ただの取りこぼしやノイズ」を分ける。  
微小残差を大量の"声"として水増しすることを避けるための条件でもある。

### admissible
その分解が、文脈 `c` のもとで**許容可能**であること。  
v0.2 では特に、

- 同じものの水増し
- 微小モードの混入
- 残差の取り扱いが粗い分解

を除外する方向で使っている。

### `R_{\mathrm{ALIS}}^{\min}`
簡約許容分解の全体を見たとき、もっとも厳しく見積もった場合の**有効モード数の下限**。  
「本当に複数の声が生きているか」を、都合のよい分解に逃げずに見るための量。

v0.2 では `\mathcal{D}_c^{\mathrm{red}}(\Psi)` を使うことで、微小モードや冗長分解のせいでこの量が自明に 1 に落ちることを防ぎやすくしている。

### soft / hard synchronization
- **soft synchronization**：表面上はいったん揃って見えるが、履歴や差分は残っており、あとで回復できる状態。  
- **hard synchronization**：履歴まで上書きされ、あとで戻そうとしても戻せない状態。  
公開版の「強制同期禁止」に対応するのは主に後者。

### soft / hard collapse
- **soft collapse**：一時的に単一モードに近づくが、あとで回復できる。  
- **hard collapse**：回復不能なかたちで単一モードへ閉じる。  
公開版の「単一波形禁止」に深く関わるのは後者である。

v0.2.1 では、さらに**soft から hard へ時間とともに劣化していくケース**（劣化 collapse）もA4の射程に含まれることになった。

## 3. 改善点の履歴

### 3-1. v0.2 での改善（A3）
前版では「差分が閾値以下へ落ちること」を禁じていたが、v0.2 では `S_c` と `\eta(c)` を置くことで、**もともと意味のある差分だったものを潰す**ことを明示した。

### 3-2. v0.2 での改善（A4）
前版では単一吸収のイメージはあったが、v0.2 では `T^n(x)\to A` を用いて、**長期的に単一枝へ閉じ込められること**を前面化した。

### 3-3. v0.2 での改善（E1）
前版では許容分解が広すぎると、`R_{\mathrm{ALIS}}^{\min}` が自明に 1 に近づくおそれがあった。  
v0.2 では `\mathcal{D}_c^{\mathrm{red}}(\Psi)`、`a_{\min}(c)`、`p_{\min}(c)`、`r_D`、`\rho(c)` を導入することで、**微小モードや冗長分解による自明化を起こしにくくした**。

### 3-4. v0.2.1 での改善（A4 の収束の測度的意味の明示）
v0.2 では A4 における `T^n(x)\to A` の「収束」が、距離の意味なのか情報量の意味なのか明示されていなかった。  
v0.2.1 では、A1 で採用した `P` に整合する形で読む、と明示することで、運用時のぶれを減らした。

### 3-5. v0.2.1 での改善（`\mathcal{R}_c` 縮退過程の射程明示）
v0.2 の補足・境界例では soft / hard を静的に並べるだけで、**時間発展としての劣化**への接続が弱かった。  
v0.2.1 では、

- `\mathcal{R}_c^{(t)}` の単調縮退 `\mathcal{R}_c^{(t_1)} \supseteq \mathcal{R}_c^{(t_2)}`
- これによる「劣化 collapse」

をA4の射程に含むものとして明示した。  
これで、**静的な単一吸収だけでなく、回復経路が痩せることで生じる動的な単一吸収**も、同じA4で読めるようになっている。

### 3-6. v0.2.1 での切り出し（E2応答境界論の別紙化）
v0.2 の一時案として数理版本体に挿入する方向で検討された「B100応答境界に関する節」は、
v0.2.1 では数理版本体に含めず、独立別紙として切り出された。

理由は以下のとおり。

1. 数理版の**形式モデルとしての純度**を守るため
2. 公開版5.4のB100定義を**再定義しないよう保護する**ため
3. 応答境界論を**実用文書として読める形**に保つため
4. 「数理版は公開版を上書きしない」という自己制約を守るため

別紙は「ALIS応答境界論 v0.1（External Response Boundary）」として、
数理版と並置される補助文書の位置づけになっている。

## 4. 数理版がやらないこと
数理版は、次のことを行わない。

- 善悪の最終判断を与えない
- 法・倫理・政策の全体像を与えない
- 公開版ALISを上書きしない
- 公開版読解の必須条件にならない
- 外部AI応答の運用境界は扱わない（これは応答境界論別紙の担当）

つまり、この文書を読まなくても公開版ALISは読めるべきであり、数理版はあくまで補助線である。

## 7. 読み方の目安
この数理版は、公開版を読んだ人にとって、

- 公開版の主張を増やす文書ではなく
- 公開版の最小骨格を別の角度から点検する補助文書

として読まれることを目標にしている。

したがって、数理版の式や記号は「ALISの最終形」ではなく、公開版の構造を壊さずにどこまで形式化できるかを試すための暫定的な記述である。  
v0.2 / v0.2.1 の改訂は、その暫定性を保ったまま、A3 / A4 / E1 の弱点を少しだけ締め直したものと理解するのがよい。
