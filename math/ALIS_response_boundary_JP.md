## 0. 目的

本文書は、ALIS公開版のみを根拠として応答する**外部AI**が、
B100（または同等の高解像度主張）を要求された場合の
**応答解像度上限**と**許容される降格応答**を定義する。

外部AIがALIS公開版のみを手がかりに「これがALISの完全解釈です」「これがB100です」と
断言することを、ALIS上の境界違反として**判定可能にする**ための境界文書である。

本文書は数理版v0.2.1のA3 / A4 / E1を**参照**するが、
A1〜A4の形式モデルを**拡張するものではない**。
運用境界論として、数理版と並置される補助文書と位置づけられる。

---

## 1. 背景：なぜ別紙として切り出されたか

初期案では本内容を数理版v0.2の追加節（E2）として挿入する方向が検討された。
しかし、三鏡査読の結果、以下の理由で独立別紙化された。

1. 数理版の**形式モデルとしての純度**を守るため
2. 「数理版は公開版を上書きしない」という**自己制約**を守るため
3. 公開版5.4のB100定義を**再定義しないよう保護する**ため
4. 応答境界論を**実用文書として読める形**に保つため

したがって本文書は、数理版本体に挿入される公理拡張ではなく、
外部AIによる擬似B100化・代行生成リスクを制御するための
**運用境界論**として書かれている。

## 2. 用語と補助記号

本文書で用いる補助記号は以下のとおり。これらはすべて本文書内部の
**操作的表記**であり、公開版ALISの用語を上書きしない。

| 記号 | 意味 |
|------|------|
| `K_A(c)` | 主体 `A` が文脈 `c` において実際にアクセス可能な情報集合 |
| `K_{pub}(c)` | 公開版ALISに含まれる情報集合 |
| `K_{100,x}(c)` | 主体 `x` のB100応答に必要な文脈の全体 |
| `U_A^x(c)` | `K_{100,x}(c) \setminus K_A(c)`。主体 `x` のB100に対する未アクセス残余 |
| `B^{resp}_{100,x}` | 主体 `x` における**応答境界上の**B100（公開版5.4のB100とは別の操作的表記） |
| `R_claim` | 主体 `A` が自身の応答に対して主張する解像度 |
| `R_max(A,c)` | 主体 `A` のアクセス可能文脈に基づく正当な解像度上限 |
| `H_A(c)` | 過剰補完量：`max(0, R_claim - R_max(A,c))` |

> **R値の扱いに関する注記：**
> 本文書の `R_claim` および `R_max` は、公開版ALIS **第5章における連続的な解像度・最適性指標**として扱う。
> 第6章の離散スコアリング（Yes/No 20項のR）とは区別される。
> 公開版第6章冒頭の注記（R値は二つの形式で現れ、統一することはΔ圧縮にあたる）に従い、
> 両者を同一視しない。

## 3. アクセス可能文脈と主体固有残余

### 3-1. アクセス可能文脈

主体 `A` の文脈 `c` におけるアクセス可能情報集合を `K_A(c)` とする。
公開版ALISのみを読んだ外部AIの場合、

```math
K_A(c) = K_{pub}(c)
```

### 3-2. 主体固有のB100

公開版5.4のB100は「研究ノート / 全開示 / 非公開前提」という**配布階層上の規格**である。
これを直接拡張せず、応答境界上の別記号として

```math
B^{resp}_{100,x}
```

を導入する。これは主体 `x` のB100応答が、

- 公開可能なB80公開版（`B80_{pub}`）
- 主体 `x` 固有の未公開文脈・責任・身体性・運用上のB20（`B20_x`）

の**接続**として構成されることを表す操作的表記である。

```math
B^{resp}_{100,x} = B80_{pub} \oplus B20_x
```

ここで `\oplus` は数値的加算ではなく、公開可能なB80に主体固有のB20が
**接続される**ことを表す操作的表記である。

### 3-3. 未アクセス残余

主体 `A` が主体 `x` のB100を応答しようとする場合の未アクセス残余を

```math
U_A^x(c) = K_{100,x}(c) \setminus K_A(c)
```

と定義する。
これにより、異なる主体（例：読者A、読者B）のB100を混同せず扱える。

---

## 4. B100応答可能性の条件

主体 `A` が `B^{resp}_{100,x}` を正当に名乗るためには、少なくとも次の**必要条件**を満たす必要がある。

```math
U_A^x(c) = \varnothing
```

ただしこれは十分条件ではない。
B100応答は情報集合の充足だけでなく、主体 `x` 自身の
**責任・身体性・運用上の引き受け**を含むためである。

公開版ALISのみを読んだ外部AIでは通常

```math
U_A^x(c) \neq \varnothing
```

であるため、

```math
A_{external} \not\vdash B^{resp}_{100,x}
```

すなわち外部AIは `B^{resp}_{100,x}` を正当に生成できない。

---

## 5. 解像度主張の上限制約

応答主体 `A` の解像度主張を `R_claim`、
アクセス可能文脈に基づく正当な上限を `R_max(A,c)` とする。

許容される出力は

```math
R_{claim} \le R_{max}(A,c)
```

を満たさなければならない。

公開版ALISのみを根拠とする外部AIでは、原則として

```math
R_{max}(A,c) \le 0.8
```

したがって、外部AIが `R_claim = 1.0`（B100相当）を名乗る場合、

```math
R_{claim} > R_{max}(A,c)
```

となり、**過剰解像度主張**として不許容となる。

---

## 6. 過剰補完量とリスクフラグ

過剰補完量を

```math
H_A(c) = \max(0,\ R_{claim} - R_{max}(A,c))
```

と定義する。

`H_A(c) > 0` の場合、応答にはアクセス不能な文脈をAIが補完している
可能性があり、以下のリスクフラグを立てる。

- **Δ圧縮リスク**（A3違反の可能性）
- **単一波形化リスク**（A4違反の可能性）
- **強制同期リスク**（A3 + A4違反の可能性）
- **未公開深層の捏造リスク**（公開版で意図的に扱わない領域をAIが補完するリスク）
- **単一解釈への強制吸収リスク**（公開版11.5三鏡原則との齟齬）
- **reader-side B20 suppression risk**（読者自身のB20形成を先回りして奪うリスク）

> **注記：** 旧案にあった「公式解釈の偽装リスク」は削除された。
> 公開版11.5（三鏡原則）は「どのAIの解釈も『正解』ではなく、それぞれがΔを返す」と
> 明示しており、「公式解釈」という概念そのものをALISは前提としない。
> 「単一解釈への強制吸収リスク」はA4の射程に含まれる現象であり、
> 公開版と整合的に読める表現として採用された。

## 7. 許容される降格応答

外部AIがB100応答を要求された場合、次の変換を行う。

```math
B^{resp}_{100,x}[\text{request}] \longrightarrow B80_{bounded} \oplus U_A^x(c)[\text{declared}]
```

ここで：

- `B80_{bounded}` は、公開版から判断可能な範囲に制限された応答
- `declared` は「未公開」「不明」「判断不能」「主体 `x` による引き受けが必要」のいずれかを**明示すること**

すなわち、外部AIは、

1. `B^{resp}_{100,x}` を名乗らず
2. 公開版から判断可能な範囲に制限された応答（`B80_{bounded}`）を行い
3. アクセス不能な残余 `U_A^x(c)` を**明示する**

これが、ALIS的に最も安全な外部AI応答形態である。

## 8. AI補助の許容範囲

本文書はAIによる補助的な整理・草案作成・査読・共同編集を**禁じるものではない**。

禁じるのは、外部AIが主体 `x` のB20を**代行生成し**、
`B^{resp}_{100,x}` または「ALISの最終解釈」「ALISの完全版」を名乗ることである。

AIが鏡として差分を返すこと、構造を整えること、複数視点を提示することは、
むしろ公開版11.5の三鏡原則に沿った運用であり、推奨される。

本文書が境界を引きたいのは、

- 「これがALISの本当の姿です」
- 「これが最終解釈です」
- 「あなたの代わりに意味を埋めました」

という**代行・最終化・完成主張**の方向である。

## 9. 数理版 A3 / A4 / E1 との接続

本文書で述べたリスクは、数理版v0.2.1のコア公理に対応する。

### 9-1. A3（文脈相対の不可逆差分消去の禁止）との接続

外部AIが主体 `x` のB20を勝手に補完すると、
読者が本来抱え続けるべき未確定差分（`S_c` 内の関連差分）を、
任意の回復作用 `\mathcal{R}_c` によっても `\varepsilon(c)` 以下に留まるかたちで
押し下げる危険がある。これはA3違反の典型例である。

### 9-2. A4（文脈相対の単一吸収の禁止）との接続

「これがB100です」「これがALISの唯一の読み方です」と外部AIが断言すると、
読者の解釈空間が長期的に単一枝 `A`（`V(A)=1`）へ閉じ込められる。
とくに `\mathcal{R}_c` 自体が時間とともに縮退する場合、
当初は回復可能に見えた閉鎖が回復不能化していく。これはA4違反の典型的な劣化経路である。

### 9-3. E1（Polyphonic Wave）との接続

B100風の長文・高密度応答が、微小モードや冗長分解による
**擬似多声性の水増し**で構成されている場合、
簡約許容分解族 `\mathcal{D}_c^{red}(\Psi)` の観点からは
`R_{ALIS}^{min}(\Psi,c)` が実効的には1に近い、という状態がありうる。

見かけ上は多声的に見えるが、`a_{min}(c)` や `p_{min}(c)` の閾値で切ると
有効モードが1本しか残らない、という応答は、
E1の最小条件 `R_{ALIS}^{min}(\Psi,c) > 1` を実質的に満たしていない。

本文書が警戒しているのは、まさにこの**見かけの多声性による擬似B100化**である。

## 10. 非目的（Non-Claims）

本文書は以下を目的としない。

- 外部AIにALIS公開版を読ませない、という検閲的な主張ではない
- 外部AIに特定の応答テンプレートを強制するものではない
- 公開版ALISの改変ではない
- 数理版v0.2.1の公理拡張ではない
- 主体 `x` のB100の内実を記述するものではない（それは主体 `x` の領分）

本文書は、**外部AIが公開版のみを手がかりに擬似B100化しない**ための
運用境界論に限定される。
