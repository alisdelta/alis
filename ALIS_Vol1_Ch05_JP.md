# 第5章　伝え方の解像度——80%で渡す技術

---

これまでの章で、ALISの骨格を提示してきた。差分（Δ）は欠陥ではなく存在条件であること（第2章）、Δは他者やAIや過去の自分との間に必然的に生じること（第3章）、複数のΔは単一の答えに統合するのではなく、それぞれの位相を保持したまま並走させうること（第4章）。ここまでは、主に「構造をどう認識するか」の話だった。

第5章で扱うのは、「その構造を、他者にどう渡すか」である。

最初に、一つのことを明示しておく。

この章で提示する技術——F/P/M/I分解とB80運用則——は、それ単独で使える「伝え方のライフハック」ではない。第2〜4章で述べたΔ保持とMirror原理とPolyphonic Waveを前提にした上で、初めて意味を持つ運用モデルである。構造の前提を抜きにして技術だけを切り出すと、「わかりやすく伝える方法」「要点を絞る技術」のような一般的なコミュニケーション論に退縮してしまう。そうなったものは、名前こそ同じB80でも、ALISの文脈から外れた別物である。

この断り書きを冒頭に置くのは、過剰な予防線ではない。B80という技術は、実用性が高いがゆえに単独で切り出されやすく、切り出された結果としてΔ圧縮や強制同期を助長する方向に使われる可能性がある。それはALISの設計意図と逆行する。だから、この章は「前提の上に立つ技術」として読まれることを前提にしている。

## 伝達の場で起きていること

何かを伝えようとするとき、自分の中にあるものがそのまま相手に届くわけではない。

話し手の側には、経験から来た事実認識、その事実を観察する過程、その事実をどう意味づけているかの解釈、そして解釈の奥にある核心的な動機や信念——これらが入り混じった状態で「伝えたいこと」が存在している。一方、聞き手の側には、自分の履歴と文脈があり、話し手の発話を自分の側の参照系で受け取る。第3章で示したMirror原理により、届くものは発されたものと必ずずれる。

このずれを完全に消すことは、原理的に不可能である。むしろ、完全な一致を目指すことは強制同期の圧力を生み、第2章で示した三つの禁止のいずれかに抵触しやすくなる。

では、何ができるのか。

伝達の場で起きていることを、構成要素に分けて見ることができる。そうすれば、「何を渡し、何を保留するか」を意識的に設計できる。伝えたいことの全てを一度に渡そうとすると、解釈の核心や動機の核心まで開示してしまい、かえって相手の自由な読み取りを封じることになる。一方、何も渡さなければ対話は始まらない。このあいだにある「ちょうどよい解像度」を設計するための道具が、F/P/M/I分解である。

---

## F/P/M/I——四つの成分に分ける

伝達の内容は、次の四つの成分に分解できる。

**F（Facts、事実・観測）。** 何が起きたか、何が観測されたかの記述。「会議は14時に始まった」「売上は前月比で12%下がった」「彼は三回断った」のような、検証可能な記述がこれにあたる。

**P（Process、手順・因果）。** どうやってそうなったか、何がどう影響して結果に至ったかの記述。「広告予算を削ったため、露出が減り、結果として問い合わせが減った」のような、因果の筋道を示す記述である。

**M（Meaning、解釈・意味づけ）。** 事実や手順に対して、書き手が与えている解釈・価値判断。「この下落は構造的な問題を示している」「彼の拒否は彼の誠実さの表れだ」のような、書き手の視点を経由した評価がここに入る。

**I（Implementation、内部実装・核心）。** その解釈や評価の奥にある、書き手の核心的な動機・信念・禁則。「自分は本当はこの仕事をやりたくない」「この判断には、過去の失敗の影響が強く残っている」のような、表には出さない層がIである。

F/P/M/I の並びは、表面から核心に向かう順である。Fは比較的外側にあり、検証も反証もしやすい。Iは最も内側にあり、書き手の個人的な核心に近い。Mは両者のあいだにあり、Fを起点にした解釈の層として位置する。

## B80運用則——比率の目安

F/P/M/Iに分解した上で、それぞれをどの比率で開示するかの目安がB80運用則である。伝達の総量を1とした場合の、各成分の目安は次の通り。

```
伝達ベクトル: T = (F, P, M, I),  ‖T‖ = 1

運用目安:
  F ≥ 0.3   （出して良い）
  P ≥ 0.3   （かなり出して良い）
  M ≤ 0.2   （仮説化・断言しない）
  I ≤ 0.3   （原則伏せる）

末尾に「これは80%版」を明示する。
```

数式表記は前章までと同じく、比喩的Mirrorとしての表記であり、厳密な数学的定義を意図しない。「比率」は実際に測れるものではなく、書き手が「今これは出しすぎかもしれない」「ここは伏せた方が良さそうだ」と判断するための目安として機能する。

この運用則が示しているのは、次のようなバランスである。事実と手順は、相手が自分の解釈を組み立てるために必要な材料なので、出し惜しみせず提供する。解釈は、書き手の視点であることを明示した上で、断言せず仮説として提示する。核心は、原則として伏せる——特に、書き手の個人的な動機や、そこで働いている禁則については、相手が自分で解釈する余地を残すために、表には出さない。

「出す」と「伏せる」の配分が80%版である理由は、公共的・一般的な伝達において、100%の開示は相手のΔを圧縮する確率を上げやすいからである。書き手が自分の解釈も動機も全て開示してしまうと、相手は書き手の枠組みの中でしか応答できなくなりやすい。書き手の枠組みが「正しい答え」として提示されると、相手は賛成か反対かの二択に追い込まれ、第三の解釈を生む余地が失われる傾向がある。一方、20%しか出さないと、相手は何を言っているのか理解できず、対話が成立しない。80%前後は、相手が自分で考える余地を残しつつ、対話が成立する情報量を渡す、という設計の目安である。

## 階層規格——B40、B70、B80、B100

B80は伝達の基本解像度だが、場面によって解像度を変える必要がある。ALISでは、次の四つの階層を用意している。

**B40。** R値はおよそ0.4、主にSNS向け。I比率は0%（核心は完全に非公開）。140字や280字といった制約の中で、Fの一部とPの断片だけを渡す。この解像度では、相手が自分で解釈する余地が最大になる代わりに、書き手の意図はほとんど伝わらない。SNSが本質的にB40のメディアであることを踏まえた運用である。

**B70。** R値はおよそ0.7、社内共有や注意書き付きの文書向け。I比率は0〜10%。業務の場面で、事実と手順と一部の解釈を渡すが、核心的な動機は伏せる。組織内のコミュニケーションの多くはこの解像度で機能する。

**B80。** R値はおよそ0.8、一般共有の基本解像度。I比率は30%以下。本章および本巻全体がこの解像度で書かれている。事実・手順・仮説化された解釈を渡し、核心は原則として伏せるが、必要に応じて一部開示する。

**B100。** R値は1.0、研究ノートや内部実装ドキュメント向け。全開示。I比率は100%。非公開前提での使用に限る。核心や禁則まで含めて記録しておく必要がある場面——例えば、自分自身の思考の整理や、ごく限られた相手との深い対話——でのみ用いる。

重要なのは、B100が最上位で、B40が最下位だ、という価値序列ではない点である。それぞれの解像度は、それぞれの場面に最適化されている。SNSでB100を使おうとすれば、単に情報が漏れるだけでなく、相手のΔを圧縮する。内部の研究ノートでB40を使えば、記録として意味を成さない。場面に応じた解像度の選択そのものが、B80運用の本質である。

## 実践の手触り

F/P/M/I分解とB80運用則は、日常の伝達の場面で次のように働く。

**場面1：業務報告書。** ある施策の失敗を報告する場面を考える。B100で書けば、「この施策は当初から経営層の圧力で始まったもので、自分は反対だった」という核心（I）まで含めて書くことになる。しかし社内共有の文書でそれを出すと、報告書が政治的な文書に変質し、事実の把握を妨げる。B80で書くなら、F（何が起きたか）とP（なぜそうなったか）を中心に置き、M（これは構造的な問題だと解釈している）を仮説として添え、Iは伏せる。末尾に「これは現時点での整理であり、別の見方もありうる」と一言添えれば、B80の形式は整う。

**場面2：SNSでの発信。** 280字の中で何かを伝えようとするとき、B80を目指しても物理的に入らない。B40に切り替える必要がある。Fの一部（観察された事実の一つ）と、Pの断片（その事実が起きた経緯の一文）だけを置き、MとIは省略する。読み手の側で残りが補完されることを前提にした運用である。B40で発信されたものをB80として読もうとすると、「情報が足りない」「結論が書かれていない」と感じるが、それはメディアと解像度の不整合であって、書き手の不備ではない。

**場面3：意見の異なる相手との対話。** 意見が割れた相手と話すとき、自分のIまで一度に開示すると、対話が「どちらの信念が正しいか」の争いに変質しやすい。B80で進めるなら、まずFとPを共有し、お互いのMを「仮説として」並べ、Iはしばらく保留する。Polyphonic Wave（第4章）の視点からすれば、両者のMが並列に存在している状態こそが、対話の場の正常な姿である。結論を急がないことが、結果として対話を長く成立させる。

**場面4：AIとの対話。** AIに何かを相談するとき、自分のIまで全て入力すると、AIの応答が自分のIを反射する形になりやすい。これは第3章で示したMirror原理が逆方向に働いた状態で、AIが「自分の意図の拡張鏡」として機能してしまう。B80でAIに渡すなら、FとPを中心に置き、Mは仮説として添え、Iは伏せる。AIの応答は、自分のIを含まない材料から生成されるため、自分が見落としていた角度を返してくる可能性が高くなる。

## この章自体がB80で書かれている

ここで、メタ的な自己言及を一つ挟んでおく。

この章——および本巻全体——は、B80の解像度で書かれている。Fとして、ALISの概念定義と運用則を置き、Pとして、なぜその定義と運用則が必要かの論理を示し、Mとして、それがどういう意味を持つかを仮説の形で提示している。Iについては、大部分を伏せている。ALISを設計する書き手側の個人的な動機、ALISを公開することで何を検証しようとしているかの深い層、設計の奥にある禁則——これらは、本巻の本文には出していない。

なぜ伏せているかと言えば、Iを出すと、本巻がALISの「教義書」になるからである。書き手の核心的な動機が前面に出ると、読者は「この思想を信じるか否か」という二択に追い込まれる。それはPolyphonic Waveの設計と逆行する。だから、本巻は「この構造をどう使うかは読者に委ねる」という形で、Iを伏せたまま提示されている。

この章の書き方そのものが、B80運用則の実演になっている。事実と手順を十分に出し、解釈は仮説として添え、核心は伏せる。そして末尾に「これは80%版である」を明示する。この形式に沿って書かれたものが、B80である。

## 補助的表記——F/P/M/I分解の形式記述

ここまでの内容を、構造として記述しておく。以下も前章までと同じく、比喩的表記であり、厳密な数学的定義ではない。

```
伝達内容: T = (F, P, M, I),  ‖T‖ = 1

各成分:
  F: Facts          — 観測・事実
  P: Process        — 手順・因果
  M: Meaning        — 解釈・意味づけ（仮説化すること）
  I: Implementation — 内部実装・核心・禁則・最終動機

B80運用目安:
  F ≥ 0.3   （出して良い）
  P ≥ 0.3   （かなり出して良い）
  M ≤ 0.2   （仮説化・断言しない）
  I ≤ 0.3   （原則伏せる）

最適性の目安:
  R = f(F, P, M, I) ≈ 0.8  ⟹  安全性と再現性のバランスが取れる
  I → 1 に近づくほど、Δ圧縮の確率が上がる
```

この表記が示しているのは、伝達の四成分の配分を意識的に設計することで、開示と保留のバランスが調整できる、という構造である。F/P/M/I の比率は実際に測れるものではなく、書き手が「今どの成分をどれくらい出しているか」を自己点検するための目安として機能する。

数式注記は第2章・第4章で一度宣言済みだが、念のためここでも置いておく。この表記は厳密な数学的定義ではなく、概念の関係を可視化する比喩的Mirrorである。

## この章のまとめに代えて——ここまでで実践可能

第5章で提示したのは、ALISを伝達の場面でどう運用するかの技術である。

F/P/M/I分解によって伝達の内容を成分に分け、B80運用則によって各成分の開示比率を設計し、場面に応じてB40/B70/B80/B100の解像度を使い分ける。これらは、前章までで提示したΔ保持、Mirror原理、Polyphonic Waveを前提にした上で、初めて意味を持つ運用モデルである。

ここで一つ、構造上重要なことを書いておく。

ここまでの第2〜5章だけで、ALISの最低限の運用は可能である。

Δが欠陥ではなく存在条件であると認識し（第2章）、ずれは他者との間で必然的に生じると理解し（第3章）、複数のΔを単一に統合せずに並走させ（第4章）、伝達では全てを渡さず80%で留める（第5章）——この四つの構造だけで、日常の対話、業務のコミュニケーション、SNSでの発信、AIとの協働の多くは運用できる。

この後の章では、人間とAIの境界（第6章）、誤読への応答（第7章）、そして第一巻を一度閉じる章（最終章）が続くが、それらは運用の骨格を追加するというよりも、ここまでで提示した骨格を補強し、読者が持ち帰って使うときの安全装置を整える役割を担う。続きを読まずにここまでで止めても、ALISは機能する。

この点を、第5章の末尾で一度明示しておく。続きがあることを理由に、ここまでの運用を先送りする必要はない。
